## Cross-Phase Integration (22/23 wired)

### Integration Issues

**INT-01: Action response missing fragmentPath/writable** (cosmetic)
- **From:** Phase 2 (services.js action handler)
- **To:** Phase 5 (ServiceRow user badge, UnitFile writable guard)
- **Issue:** `POST /api/services/:name/action` response builds service object from local `SHOW_PROPS` that omits `FragmentPath`. After any action, the affected row's `service.writable` becomes `undefined`, causing the "user" badge to disappear for 0-10 seconds until the next polling cycle restores full data.
- **Fix:** Add `FragmentPath` to `SHOW_PROPS` in `services.js` and compute `writable` in the action response.
- **Severity:** Cosmetic/transient. The UnitFile page is unaffected (reads from its own `GET /api/unit/:service` fetch).

### Inefficiencies (non-blocking)

- **Double `/api/system` fetch:** `Layout.tsx` and `Home.tsx` both independently fetch `GET /api/system` on mount. Could share via React context.

All 23 requirements have verified integration paths from backend through API to frontend. See integration checker report for full wiring map.
