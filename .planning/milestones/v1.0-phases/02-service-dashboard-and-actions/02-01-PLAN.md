<task type="auto">
  <name>Task 1: Create systemctl parsing utility and service/system API routes</name>
  <files>server/utils/systemctl.js, server/routes/services.js, server/routes/system.js</files>
  <action>
Create three new server files. All are plain JS (ESM, not TypeScript) per project convention.

**server/utils/systemctl.js** — Parsing helpers and data fetching:
- `parseListUnits(stdout)`: Split on newlines, filter empty, split each line on `\s+`. Return array of `{ unit, load, active, sub, description }`. Description is `parts.slice(4).join(' ')` (it contains spaces — Pitfall 3 from research).
- `parseShowOutput(stdout)`: Split on `\n\n` for blocks, each block split on `\n` for `key=value` lines (split at first `=` only). Return a `Map<Id, props>`.
- `getAllServices()`: Run `runSystemctl('list-units', null, ['--all', '--type=service', '--plain', '--no-legend'])` and `runSystemctl('show', null, ['--type=service', '-p', SHOW_PROPS])` in parallel with `Promise.all`. SHOW_PROPS = `'Id,Description,MainPID,MemoryCurrent,CPUUsageNSec,ActiveEnterTimestamp,ActiveState,LoadState,SubState,UnitFileState'`. Merge by service name: list-units is source of truth for the full list (173 services), show data fills in details where available. Guard `MemoryCurrent` and `CPUUsageNSec` for `[not set]` (return null). Guard empty `ActiveEnterTimestamp` (return null). `MainPID` of `'0'` becomes null. All numeric values parsed with `parseInt()`.

Import `runSystemctl` from `'../utils/exec.js'` (existing wrapper, no changes to exec.js).

**server/routes/services.js** — Express router:
- `GET /` (mounted at /api/services): Call `getAllServices()`, return `{ ok: true, services }`. Wrap in try/catch, pass errors to `next(err)`.
- `POST /:name/action`: Read `action` from `req.body`. Validate action is one of `['start', 'stop', 'restart', 'enable', 'disable']` — return 400 if not. Call `runSystemctl(action, name)`. Then re-fetch updated state for that single service: `runSystemctl('show', name, ['-p', SHOW_PROPS])`, parse with `parseShowOutput`, extract the service object. Return `{ ok: result.ok, stderr: result.stderr || undefined, service: { ...updatedFields } }`. The exec.js wrapper handles service name validation (rejects invalid names with an error), so the route does not need to re-validate the name regex.

**server/routes/system.js** — Express router:
- `GET /` (mounted at /api/system): Return `{ ok: true, hostname: os.hostname(), uptimeSeconds: Math.floor(os.uptime()) }`. Import `os` from `'node:os'`.
