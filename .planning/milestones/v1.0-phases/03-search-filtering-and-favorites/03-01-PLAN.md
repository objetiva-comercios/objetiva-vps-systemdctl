<task type="auto">
  <name>Task 1: SQLite watched_services table and REST toggle routes</name>
  <files>server/db.js, server/routes/watched.js, server/index.js</files>
  <action>
1. In `server/db.js`, add after the existing `db.pragma('foreign_keys = ON')` line:
   ```javascript
   db.exec(`
     CREATE TABLE IF NOT EXISTS watched_services (
       unit TEXT PRIMARY KEY NOT NULL,
       added_at INTEGER NOT NULL DEFAULT (unixepoch())
     )
   `)
   ```
   This is idempotent — safe to run on every server start. `unixepoch()` is supported in SQLite 3.38.0+ (installed is 3.51.2).

2. Create new file `server/routes/watched.js`:
   - Import express and db from '../db.js'
   - Define `UNIT_NAME_RE = /^[\w@\-.:\/\\]+\.service$/` — wider than exec.js SERVICE_NAME_RE because watched route never calls systemctl, and some unit names contain backslash (e.g. `systemd-fsck@dev-disk-by\x2dlabel-BOOT.service`). Per research Pitfall 3.
   - Prepare statements at module level (not inside handlers): `add` (INSERT OR IGNORE), `remove` (DELETE WHERE unit = ?)
   - POST `/:name` — validate name against UNIT_NAME_RE, run `stmts.add.run(name)`, return `{ ok: true, unit: name, isWatched: true }`
   - DELETE `/:name` — validate name against UNIT_NAME_RE, run `stmts.remove.run(name)`, return `{ ok: true, unit: name, isWatched: false }`
   - Invalid names return 400 with `{ ok: false, error: 'Invalid unit name' }`
   - Export default router

3. In `server/index.js`, add the watched router mount:
   - Import: `import watchedRouter from './routes/watched.js'`
   - Mount: `app.use('/api/watched', watchedRouter)` — add after the existing `app.use('/api/system', systemRouter)` line, before the error middleware
  </action>
