**src/types/service.ts** — TypeScript interfaces:
```typescript
export interface ServiceEntry {
  unit: string
  load: string
  active: string        // 'active' | 'inactive' | 'failed' | 'activating' | 'deactivating'
  sub: string           // 'running' | 'dead' | 'exited' | 'failed' | 'waiting' | etc.
  description: string
  unitFileState: string // 'enabled' | 'disabled' | 'static' | 'masked' | ''
  pid: number | null
  memoryBytes: number | null
  cpuNsec: number | null
  activeEnterTimestamp: string | null
}

export interface SystemInfo {
  hostname: string
  uptimeSeconds: number
}
```

Also export formatting helper functions in this same file:
- `formatBytes(bytes: number | null): string` — null returns `'--'`, then KB/MB/GB thresholds
- `formatCpuTime(nsec: number | null): string` — null returns `'--'`, convert to ms/s/m format
- `formatUptime(timestamp: string | null): string` — null returns `'--'`, parse timestamp (strip day-of-week prefix: `parts.slice(1).join(' ')`), compute seconds since then, format as s/m/h/d
- `formatSystemUptime(seconds: number): string` — format d/h/m

Use `'--'` (double dash) not em-dash for null values — monospace font renders it cleanly.

**src/hooks/useServicePolling.ts** — Polling hook:
- State: `services: ServiceEntry[]`, `loading: boolean` (true initially), `error: string | null`, `lastUpdated: Date | null`
- Use a `useRef<boolean>` for `isFetching` guard to prevent overlapping fetch calls (Pitfall 5 from research)
- `fetchServices` callback: if `isFetching.current` is true, return early. Set `isFetching.current = true`, fetch `/api/services`, parse JSON, update state, set `isFetching.current = false` in finally block.
- `useEffect`: call `fetchServices()` immediately, then `setInterval(fetchServices, 10_000)`. Return cleanup that calls `clearInterval`.
- Export: `{ services, setServices, loading, error, lastUpdated, refresh: fetchServices }`
- Expose `setServices` so the parent can optimistically update a single row after an action without waiting for the next poll cycle.

**src/components/ServiceRow.tsx** — Single row with actions:
- Props: `service: ServiceEntry`, `onServiceUpdate: (updated: ServiceEntry) => void`
- State: `actionPending: string | null` (tracks which action is in progress)
- State: `actionError: string | null` (shows error message, auto-clears after 3 seconds)
- `handleAction(action: string)`: Set `actionPending`, POST to `/api/services/${encodeURIComponent(service.unit)}/action` with `{ action }`, on success call `onServiceUpdate(data.service)` to update parent state, on failure set `actionError`. Always clear `actionPending` in finally.
- Render a table row `<tr>` with columns:
  1. **Service name**: `service.unit` in monospace, truncate long names with `truncate` class and `max-w-xs`
  2. **Status**: Active state color-coded with small icon: `CircleCheck` green for active, `CircleAlert` red for failed, `CircleDot` muted for inactive. Show sub-state text next to icon.
  3. **Enabled**: Badge showing `unitFileState` — green outline for `enabled`, muted outline for `disabled`, plain text for `static`/`masked`/empty.
  4. **PID**: `service.pid ?? '--'`
  5. **Memory**: `formatBytes(service.memoryBytes)`
  6. **CPU**: `formatCpuTime(service.cpuNsec)` — column header says "CPU" (not "CPU %", since this is cumulative time)
  7. **Uptime**: `formatUptime(service.activeEnterTimestamp)`
  8. **Actions**: Row of icon buttons. Show based on service state:
     - If active/running: Stop (CircleStop), Restart (RefreshCw)
     - If inactive/dead/failed: Start (Play)
     - Always show: Enable (Power) if not enabled, Disable (PowerOff) if enabled
     - While `actionPending` is set, show `LoaderCircle` with `animate-spin` on the pending button and disable all buttons in the row
     - On `actionError`, briefly show error text in red (auto-clear with setTimeout 3s)
- Icon buttons: small, `p-1`, `rounded`, `hover:bg-bg-elevated`, `text-text-muted hover:text-accent` (or `hover:text-danger` for stop/disable)
