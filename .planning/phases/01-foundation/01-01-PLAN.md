<task type="auto">
  <name>Task 2: Create Express server, config, SQLite database, and execFile security wrapper</name>
  <files>
    server/index.js
    server/config.js
    server/db.js
    server/utils/exec.js
  </files>
  <action>
    Create the `server/` directory with four files. All files use ESM (`import`/`export`).

    **server/config.js** — Environment configuration with safe defaults:
    - Import `dotenv/config` (side-effect import that loads .env)
    - Export `PORT` (default 7700), `HOST` (default '127.0.0.1'), `DB_PATH` (default './data/systemdctl.db'), `NODE_ENV` (default 'development')
    - All values read from `process.env` with fallbacks
    - PORT must be parsed with `parseInt(..., 10)`

    **server/db.js** — SQLite initialization:
    - Import `Database` from `better-sqlite3` and `DB_PATH` from config
    - Use `mkdirSync(dirname(DB_PATH), { recursive: true })` to ensure data directory exists BEFORE opening the database
    - Create database instance: `new Database(DB_PATH)`
    - Set `pragma('journal_mode = WAL')` and `pragma('foreign_keys = ON')`
    - Export the db instance as default

    **server/utils/exec.js** — THE security-critical execFile wrapper (INFR-03 + INFR-04):
    - Import `execFile` from `node:child_process` and `promisify` from `node:util`
    - Create `execFileAsync = promisify(execFile)`
    - Define `ALLOWED_ACTIONS = Object.freeze(['start', 'stop', 'restart', 'enable', 'disable', 'status', 'is-active', 'is-enabled', 'show', 'list-units'])`
    - Define `SERVICE_NAME_RE = /^[\w@\-.]+$/`
    - Export async function `runSystemctl(action, serviceName = null, extraArgs = [])`:
      1. Validate action is in ALLOWED_ACTIONS — throw Error if not (INFR-03)
      2. Validate serviceName against SERVICE_NAME_RE if not null — throw Error if invalid (INFR-04)
      3. Build args array: `['--no-pager', action]`, push serviceName if present, spread extraArgs
      4. Call `execFileAsync('/usr/bin/systemctl', args, { timeout: 30_000, maxBuffer: 5 * 1024 * 1024 })` — NEVER set `shell: true`
      5. Return `{ ok: true, stdout: stdout.trim(), stderr: stderr.trim(), code: 0 }` on success
      6. Return `{ ok: false, stdout: err.stdout?.trim() ?? '', stderr: err.stderr?.trim() ?? err.message, code: err.code ?? null }` on failure
    - Also export `ALLOWED_ACTIONS` and `SERVICE_NAME_RE` for potential testing use

    **server/index.js** — Express application entry point (INFR-02):
    - Import `express` from `express`, config values from `./config.js`
    - Import `path` from `node:path` and `fileURLToPath` from `node:url`
    - Compute `__dirname` via `path.dirname(fileURLToPath(import.meta.url))`
    - Compute `DIST = path.join(__dirname, '..', 'dist')` — note the `..` because server/ is one level deep
    - Create Express app, use `express.json()` middleware
    - Import `./db.js` (side-effect: initializes database on startup)
    - Add a health check route: `GET /api/health` returns `{ ok: true, timestamp: new Date().toISOString() }`
    - Add error handling middleware (registered last before static):
      ```javascript
      app.use((err, req, res, next) => {
        const status = err.status ?? 500
        res.status(status).json({
          ok: false,
          error: err.message ?? 'Internal server error',
          details: err.details ?? undefined,
        })
      })
      ```
    - Serve static files: `app.use(express.static(DIST, { index: false }))` — `index: false` is critical
    - SPA catch-all (AFTER all API routes): `app.get('*', (req, res) => res.sendFile(path.join(DIST, 'index.html')))`
    - Listen: `app.listen(PORT, HOST, () => console.log(\`systemdctl listening on http://${HOST}:${PORT}\`))` — HOST as second argument ensures 127.0.0.1 binding (INFR-02)

    CRITICAL ORDERING in index.js:
    1. express.json() middleware
    2. API routes (/api/*)
    3. Error middleware
    4. express.static (with index: false)
    5. SPA catch-all
  </action>
