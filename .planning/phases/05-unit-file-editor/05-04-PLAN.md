  <action>
In UnitFile.tsx, replace the read-only `<pre>` block (lines 190-194) with a read-only CodeMirror instance that reuses the existing `systemdLang` extension.

Replace this block:
```tsx
{!loading && !error && unitInfo && !editing && (
  <pre className="font-mono text-xs text-text-primary bg-bg-surface border border-border rounded-md p-4 overflow-auto whitespace-pre-wrap flex-1">
    {unitInfo.content}
  </pre>
)}
```

With a read-only CodeMirror component:
```tsx
{!loading && !error && unitInfo && !editing && (
  <CodeMirror
    value={unitInfo.content}
    height="600px"
    theme="dark"
    extensions={[systemdLang]}
    editable={false}
    readOnly={true}
    basicSetup={{
      lineNumbers: true,
      foldGutter: false,
      highlightActiveLine: false,
    }}
    style={{ fontFamily: 'JetBrains Mono Variable, monospace', fontSize: '13px' }}
    className="border border-border rounded-md overflow-hidden flex-1"
  />
)}
```

Key differences from the edit-mode CodeMirror:
- `editable={false}` — prevents cursor and text selection editing
- `readOnly={true}` — additional read-only protection
- `highlightActiveLine: false` — no active line highlighting since it's not editable
- Same `systemdLang` extension, same dark theme, same font — visual consistency between read and edit modes

The CodeMirror component and systemdLang constant are already imported at the top of the file, so no new imports are needed.
