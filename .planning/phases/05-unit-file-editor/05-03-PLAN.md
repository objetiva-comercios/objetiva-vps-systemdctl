  <name>Task 1: Fix EACCES â€” replace direct fs.writeFile with sudo cp privilege escalation</name>
  <files>server/routes/unit.js</files>
  <action>
In the PUT handler (lines 112-116), replace the direct fs.writeFile + fs.rename approach with a sudo-based write:

1. Write the temp file to `/tmp/` instead of the target directory (dirname(destPath)). Use the same naming pattern: `.tmp-` + basename + random hex. The temp file path becomes `join('/tmp', '.tmp-' + basename(destPath) + '.' + randomBytes(4).toString('hex'))`.

2. Write content to the temp file using `writeFile(tmpPath, content, { encoding: 'utf8', mode: 0o644 })` â€” this succeeds because /tmp is world-writable.

3. Replace the `rename(tmpPath, destPath)` call with two `execFileAsync` calls using sudo:
   - `execFileAsync('/usr/bin/sudo', ['cp', tmpPath, destPath], { timeout: 10_000 })` â€” copies the temp file to the destination with root privileges.
   - `execFileAsync('/usr/bin/sudo', ['chmod', '0644', destPath], { timeout: 10_000 })` â€” ensures correct permissions on the destination file.

4. Clean up: `unlink(tmpPath)` after the sudo cp succeeds (wrap in try/catch so cleanup failure doesn't fail the request). Import `unlink` from `node:fs/promises` alongside existing imports.

5. Import `execFile` from `node:child_process` and `promisify` from `node:util` at the top of unit.js (or import `execFileAsync` from a shared location â€” but since exec.js already has its own internal promisified version that is not exported, create a local one in unit.js: `const execFileAsync = promisify(execFile)`).

6. Remove the `rename` import from the `node:fs/promises` import line since it's no longer used. Keep `readFile`, `writeFile`. Add `unlink`.

7. Remove the `dirname` import from `node:path` since the temp file now goes to `/tmp` (dirname is no longer needed). Keep `resolve`, `basename`, `join`.

The final write sequence in the PUT handler should be:
```
const tmpPath = join('/tmp', '.tmp-' + basename(destPath) + '.' + randomBytes(4).toString('hex'))
await writeFile(tmpPath, content, { encoding: 'utf8', mode: 0o644 })
await execFileAsync('/usr/bin/sudo', ['cp', tmpPath, destPath], { timeout: 10_000 })
await execFileAsync('/usr/bin/sudo', ['chmod', '0644', destPath], { timeout: 10_000 })
try { await unlink(tmpPath) } catch {}
```

Do NOT use `shell: true` in the execFileAsync options â€” maintain the same security posture as exec.js.
