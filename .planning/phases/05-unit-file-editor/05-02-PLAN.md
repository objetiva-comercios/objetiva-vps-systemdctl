npm install @uiw/react-codemirror @codemirror/legacy-modes @codemirror/language

**Step 2 — Create `src/types/unit.ts`:**
```typescript
export interface UnitFileInfo {
  service: string
  path: string        // FragmentPath from systemctl show
  content: string     // Full text of the unit file
  writable: boolean   // true only if path is under /etc/systemd/system/
}
```

**Step 3 — Create `src/pages/UnitFile.tsx`:**

Follow the exact same page pattern as `src/pages/Logs.tsx`:
- Default export `UnitFile` component that reads `useParams<{ service?: string }>()`.
- If no service, show a centered prompt "Select a service to view its unit file." with a back link to `/` (same pattern as Logs.tsx no-service state).
- If service present, render a `UnitFileViewer` child component.

**UnitFileViewer component (inside UnitFile.tsx):**

State:
- `unitInfo: UnitFileInfo | null` — loaded from GET /api/unit/:service
- `loading: boolean` — initial fetch loading state
- `error: string | null` — fetch error message
- `editing: boolean` — false = read-only view, true = CodeMirror editor
- `editContent: string` — current editor content (initialized from unitInfo.content when entering edit mode)
- `saving: boolean` — true while PUT is in flight
- `saveError: string | null` — save error message
- `saveSuccess: boolean` — briefly true after successful save (for feedback)

Fetch on mount:
- `useEffect` with `useCallback` pattern (same as Logs.tsx): `fetch('/api/unit/' + encodeURIComponent(service))`, parse JSON, set `unitInfo` or `error`.

**Read-only view layout (editing === false):**
- Header row: ArrowLeft back link to `/`, service name as `<h1>` (same style as Logs.tsx header), file path shown as `<span className="text-text-muted text-xs font-mono">{unitInfo.path}</span>`.
- Edit button: only shown when `unitInfo` is loaded. If `unitInfo.writable === true`: `<button>` with `Edit` icon (from lucide-react, use `Pencil` icon) + "Edit" label. If `unitInfo.writable === false`: same button but `disabled` with `title="Package-managed file — editing not allowed"` and `opacity-40 cursor-not-allowed` styling.
- Content area: `<pre className="font-mono text-xs text-text-primary bg-bg-surface border border-border rounded-md p-4 overflow-auto whitespace-pre-wrap">{unitInfo.content}</pre>` — full unit file text in a scrollable pre block.

**Edit view layout (editing === true):**
- Header row: same ArrowLeft + service name + path. Replace Edit button with Save and Cancel buttons.
- Save button: `Save` icon (lucide-react) + "Save" label. Disabled when `saving` is true OR when `editContent === unitInfo.content` (no changes). Shows `LoaderCircle animate-spin` when saving.
- Cancel button: `X` icon (lucide-react) + "Cancel" label. Resets `editing` to false and clears `editContent`.
- CodeMirror editor area:
  ```tsx
  import CodeMirror from '@uiw/react-codemirror'
  import { StreamLanguage } from '@codemirror/language'
  import { properties } from '@codemirror/legacy-modes/mode/properties'

  // Module-level constant — NOT inside the component
  const systemdLang = StreamLanguage.define(properties)
  ```
  Render:
  ```tsx
  <CodeMirror
    value={editContent}
    height="600px"
    theme="dark"
    extensions={[systemdLang]}
    onChange={(val) => setEditContent(val)}
    basicSetup={{
      lineNumbers: true,
      foldGutter: false,
      highlightActiveLine: true,
    }}
    style={{ fontFamily: 'JetBrains Mono Variable, monospace', fontSize: '13px' }}
    className="border border-border rounded-md overflow-hidden"
  />
  ```

**Save handler:**
- Set `saving = true`, `saveError = null`.
- `fetch('/api/unit/' + encodeURIComponent(service), { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ content: editContent }) })`.
- On success (`data.ok`): update `unitInfo.content` to `editContent`, set `editing = false`, flash `saveSuccess = true` for 2 seconds via `setTimeout`.
- On error: set `saveError` to `data.error || 'Save failed'`.
- Finally: `saving = false`.

**Enter edit mode handler:**
- Set `editContent = unitInfo.content`, `editing = true`, `saveError = null`.

**Styling:**
- Follow the dark terminal aesthetic: bg-bg-base, text-text-primary, border-border classes.
- Error states: text-danger with AlertTriangle icon (same as Logs.tsx pattern).
- Loading state: LoaderCircle animate-spin with "Loading unit file..." text.
- Success flash after save: brief green "Saved" text with Check icon (lucide-react) that fades after 2 seconds.
